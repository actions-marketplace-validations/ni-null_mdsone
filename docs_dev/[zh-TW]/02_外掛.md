# 外掛開發

## 設定來源（簡述）

外掛設定由框架統一整理後才進入執行階段：

```text
CLI / ENV / TOML -> 規格映射 -> 合併 config -> 外掛執行 hook
```

## Hook 分類（執行 / 輸出）

| Hook | 簽名 | 呼叫時機 | 你可做什麼 | 回傳 |
| --- | --- | --- | --- | --- |
| `isEnabled` | `(config)` | 每個 hook 前的前置判斷 | 依設定決定是否啟用外掛 | `boolean` |
| `extendMarkdown` | `(md, config, context)` | Markdown render 前 | 註冊 markdown-it 規則、調整語法處理 | `void` / `Promise<void>` |
| `processDom` | `(dom, config, context)` | 文件片段 HTML 產生後 | 用 DOM API 直接改寫節點 | `void` / `Promise<void>` |
| `getAssets`（可選） | `(config)` | 模板組裝前 | 覆寫預設資產注入清單 | `{ cssFiles?: string[]; jsFiles?: string[] }` |
| `processOutputHtml` | `(html, config)` | 完整 HTML 組裝後、寫檔前 | 全頁字串後處理（壓縮、替換） | `string` / `Promise<string>` |

### 資產注入規則（重點）

1. 外掛啟用且「未實作 `getAssets`」：框架會自動收集 `plugins/<pluginName>/assets/**/*.css|js` 並 inline。
2. 外掛啟用且「有實作 `getAssets`」：只注入 `getAssets` 回傳清單（用於 mode/條件精細控制）。

## Hook 輸入速查

| 參數 | 型別 | 說明 |
| --- | --- | --- |
| `config` | `Config` | 已合併完成的最終設定（`default < TOML < ENV < CLI`） |
| `context.sourceDir` | `string` | 當前來源資料夾，常用於相對路徑解析 |
| `context.markdownText` | `string?` | 原始 Markdown 內容（主要用在 `extendMarkdown`） |
| `context.templateData` | `TemplateData?` | 已載入模板設定，可用於依模板差異切換行為 |
| `md` | `unknown` | markdown-it 實例（通常轉型成 `MarkdownIt` 使用） |
| `dom` | `unknown` | 目前是 Cheerio DOM（可轉型成 `CheerioAPI` 使用） |
| `html` | `string` | 完整頁面 HTML 字串 |

## 各 Hook 如何使用

### 1) `isEnabled(config)`

用途：決定外掛是否參與後續流程。

```ts
isEnabled(config) {
  return (config.plugins?.config?.demo as { enable?: boolean } | undefined)?.enable === true;
}
```

### 2) `extendMarkdown(md, config, context)`

用途：在 markdown-it render 前擴充語法。

```ts
import type MarkdownIt from "markdown-it";

async extendMarkdown(mdUnknown, _config, _context) {
  const md = mdUnknown as MarkdownIt;
  md.use(/* your markdown-it plugin */);
}
```

### 3) `processDom(dom, config, context)`

用途：針對片段 HTML 做 DOM 後處理（加屬性、包裝節點、插入按鈕等）。

```ts
import type { CheerioAPI } from "cheerio";

processDom(domUnknown) {
  const $ = domUnknown as CheerioAPI;
  $("pre > code").each((_i, el) => {
    $(el).attr("data-demo", "1");
  });
}
```

### 4) `getAssets(config)`（可選）

用途：只有在你需要精細控制注入檔案時才需要（例如依 mode 切換）。

不實作（建議預設）：

```ts
// 不寫 getAssets，框架自動注入 assets/ 下 css/js
```

實作覆寫（進階）：

```ts
getAssets(config) {
  const mode = (config.plugins?.config?.demo as { mode?: string } | undefined)?.mode ?? "line";
  return mode === "line"
    ? { cssFiles: ["line.css"], jsFiles: ["copy.js"] }
    : { cssFiles: ["cmd.css"], jsFiles: ["copy.js"] };
}
```

### 5) `processOutputHtml(html, config)`

用途：處理完整頁面字串。注意這是最後階段；`minify` 仍會被框架固定在最末端執行。

```ts
processOutputHtml(html) {
  return html.replace("data-demo=\"1\"", "data-demo=\"ready\"");
}
```

## 生命週期流程

1. 系統合併設定：`default < TOML < ENV < CLI`
2. `isEnabled` 判斷是否啟用
3. `extendMarkdown` 擴充 markdown-it
4. 核心渲染 Markdown 成 HTML
5. `processDom` 處理片段 DOM
6. 資產收集
7. 若有 `getAssets`：用回傳清單
8. 若無 `getAssets`：自動收集 `assets/**/*.css|js`
9. 模板組裝完整頁面（框架自動 inline 資產）
10. `processOutputHtml` 做全頁最後處理
