# 範本主題

`mdsone` 透過「範本主題」來控制輸出 HTML 的外觀與部分行為（例如 TOC、配色、程式碼高亮主題）。

## 使用範本

```bash
npx mdsone ./docs -m --template normal
```

## 變體（variant）

範本可以定義多個變體，用 `@` 指定：

```bash
npx mdsone ./docs -m --template normal@default
npx mdsone ./docs -m --template normal@warm-cream
```

說明：

- `--template <NAME|PATH[@VARIANT]>`
- 如果未指定 `@variant`，會使用 `default`

## template.config.json

範本主題資料夾內的 `template.config.json` 會放在 `config` 區塊中：

```json
{
  "_metadata": {
    "name": "normal",
    "version": "1.1.0",
    "schema_version": "v1"
  },
  "config": {
    "palette": "fog-gray",
    "types": {
      "default": {
        "code": {
          "Shiki": {
            "dark": "github-dark",
            "light": "github-light",
            "auto_detect": true
          }
        }
      }
    }
  }
}
```

### 欄位說明

| 欄位 | 說明 |
|------|------|
| `config.palette` | 預設配色名稱 |
| `config.types` | 範本變體定義 |
| `config.types.<name>.palette` | 可選。未設定時會使用變體鍵名作為配色（`default` 仍回退到 `config.palette`） |
| `config.types.<name>.code.Shiki.dark` | 深色模式使用的 Shiki 主題代號 |
| `config.types.<name>.code.Shiki.light` | 明亮模式使用的 Shiki 主題代號 |
| `config.types.<name>.code.Shiki.auto_detect` | 當 fenced code block 沒有指定語言時，是否啟用自動判斷 |

補充：

- TOC 是否啟用與標題層級，已改由 `template.html` 內的主題腳本固定控制（`TOC_ENABLED`、`TOC_LEVELS`）

## 程式碼高亮如何運作（Shiki）

目前核心會先輸出一般的 `<pre><code class="language-xxx">...</code></pre>`，再由 `plugins/shiki` 做 HTML 後處理：

1. 若 code block 已指定語言（例如 ```js），直接用該語言高亮
2. 若未指定語言且 `auto_detect=true`，用 `highlight.js` 嘗試判斷語言
3. 若判斷失敗或語言不存在，預設以 `text` 方式輸出（不會報錯）

效能提示：

- 自動判斷只會作用在「未指定語言」的 code block；如果你的文件大多都有標註語言，效能影響通常不大
