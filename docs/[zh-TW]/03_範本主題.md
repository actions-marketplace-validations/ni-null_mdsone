# 範本主題

## 目前內建模板

目前專案內建模板為 `normal`。

```bash
npx mdsone ./docs -m --template normal
```

## 模板變體

可用 `--template <theme-or-path>@<variant>` 切換模板定義的變體。若名稱不存在，系統會自動回退到 `default`。

```bash
npx mdsone ./docs -m --template normal@default
npx mdsone ./docs -m --template normal@fog-gray
npx mdsone ./docs -m --template normal@warm-cream
npx mdsone ./docs -m --template normal@sage-mist
npx mdsone ./docs -m --template normal@soft-dusk
```

## template.config.json

模板的可調設定集中在 `template.config.json` 的 `config` 區塊。

```json
{
  "_metadata": {
    "version": "1.1.0",
    "schema_version": "v1"
  },
  "config": {
    "toc": {
      "enabled": true,
      "levels": [2, 3]
    },
    "palette": "fog-gray",
    "types": {
      "default": {
        "palette": "fog-gray",
        "code": {
          "Shiki": {
            "dark": "github-dark",
            "light": "github-light",
            "auto_detect": true
          }
        }
      }
    }
  }
}
```

### 重要欄位

| 欄位 | 說明 |
|------|------|
| `config.toc.enabled` | 是否顯示 TOC |
| `config.toc.levels` | TOC 要抓哪些標題層級 |
| `config.palette` | 模板預設配色名稱 |
| `config.types.<name>.palette` | 某個模板變體使用的配色 |
| `config.types.<name>.code.Shiki.dark` | 該變體的暗色程式碼主題 |
| `config.types.<name>.code.Shiki.light` | 該變體的亮色程式碼主題 |
| `config.types.<name>.code.Shiki.auto_detect` | 未指定語言時是否自動判斷 |

## 程式碼高亮

程式碼高亮由 `plugins/shiki` 在 `processHtml` 階段處理，主題不需要直接呼叫 Shiki API。

- 是否啟用高亮：由 CLI / env / `config.toml` 的 `code_highlight` 控制
- 使用哪組 Shiki 主題：由模板的 `template.config.json` 控制
- 未指定 fenced code 語言時：可由 `auto_detect` 決定是否交給 `highlight.js` 自動判斷語言

這表示：

1. 核心只負責先輸出一般 `<pre><code>`
2. `plugins/shiki` 會再把 code block 改寫為 Shiki 結果
3. 模板只需定義自己適合的 Shiki 主題搭配

## 語言名稱顯示

模板不需要自行維護 `locale_name_*` 對照表。

語言切換選單顯示名稱由全域檔案控制：

```text
locales/config.json
```

例如：

```json
{
  "en": "English",
  "zh_TW": "繁體中文"
}
```
