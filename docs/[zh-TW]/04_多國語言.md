# 多國語言

多國語言功能只有在 `--i18n-mode` 模式下才會啟用。
單語模式 UI 語系由 `config.toml` / 環境變數設定，不會載入多語內容資料夾。

## 檔案結構

### 全域語系檔

```text
locales/
├── config.json
├── en.json
└── zh-TW.json
```

- `en.json`、`zh-TW.json`：CLI 與全域 UI 字串
- `config.json`：語言代碼對照顯示名稱，例如語言切換器

`locales/config.json` 範例：

```json
{
  "en": "English",
  "zh_TW": "繁體中文",
  "zh_CN": "简体中文"
}
```

### 模板語系檔

模板可在自己的 `locales/` 目錄定義模板專屬字串：

```text
templates/
└── normal/
    └── locales/
        ├── en.json
        └── zh-TW.json
```

模板語系檔只需要放模板自己的字串，不需要再放 `locale_name_*`。

### 多語內容資料夾

```text
docs/
├── [zh-TW]/
│   ├── 01_intro.md
│   └── 02_usage.md
└── [en]/
    ├── 01_intro.md
    └── 02_usage.md
```

子目錄名稱必須使用 `[locale]` 格式，例如 `[zh-TW]`、`[en]`。

## 相關參數

| 參數 | 說明 | 範例 |
|------|------|------|
| `--i18n-mode [CODE]` | 啟用多國語言模式（可選 CODE 指定預設顯示語系；指定時僅支援 `--i18n-mode=CODE`） | `--i18n-mode=zh-TW` |

```bash
# 多語模式會自動走合併流程，輸出單一 HTML
npx mdsone ./docs --i18n-mode=zh-TW -o dist/index.html
```

## 環境變數（shell / Docker / CI 注入）

```env
I18N_MODE=true
DEFAULT_LOCALE=zh-TW
LOCALE=zh-TW
```

> 不會自動讀取 `.env`、`.env.local`；只讀取執行當下的 `process.env`。

## config.toml

```toml
[i18n]
mode = true
default_locale = "zh-TW"
locale = "zh-TW"
```

## 輸出行為

- 產生的 HTML 內建語言切換選單
- 切換語系時頁面即時更新，不需重新整理
- 使用者選擇的語系會寫入 `localStorage`
- `html lang` 來自對應語系的模板字串 `template.html_lang`

## 預設邏輯

- `i18n.mode` 預設為 `false`
- `i18n.default_locale` 預設為空字串
- i18n 模式下若未指定 `default_locale`，會依序嘗試：
  `config.default_locale` → `config.locale` → 第一個可用語系 → `en`
